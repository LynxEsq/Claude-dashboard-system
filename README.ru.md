# CSM — Claude Session Manager

Система мониторинга и управления несколькими сессиями [Claude Code](https://docs.anthropic.com/en/docs/claude-code) в tmux в реальном времени. Управляйте всеми AI-сессиями с единого веб-дашборда.

## Обзор

CSM определяет статус каждой сессии Claude Code (Working, Needs Input, Idle, Error), анализируя вывод панелей tmux в реальном времени, и отображает всё через веб-дашборд с WebSocket-обновлениями. Также предоставляет пайплайн задач — от идей (inbox) через планирование к автоматическому выполнению.

### Основные возможности

- **Мониторинг сессий** — опрос панелей tmux каждые 3 секунды, определение состояния Claude через паттерны
- **Веб-дашборд** — многоколоночный SPA с панелями проектов, входящих, задач и терминала
- **WebSocket-обновления** — мгновенные уведомления о смене статуса, алертах и событиях задач
- **Пайплайн задач** — рабочий процесс: Желания → Планирование → Задачи → Выполнение
- **Отслеживание токенов** — использование токенов в реальном времени с прогресс-барами и алертами
- **Управление сессиями** — создание, завершение, фокусировка и отправка ввода прямо из дашборда
- **Алерты** — настраиваемые таймауты для состояний «Needs Input» и «Idle»
- **Интеграция с tmux** — переключение фокуса одним кликом, рекомендуемая конфигурация tmux

## Дашборд

Веб-интерфейс доступен по адресу `http://localhost:9847` и содержит:

| Колонка | Описание |
|---------|----------|
| **Projects** | Карточки сессий с живым статусом, использованием токенов и быстрыми действиями |
| **Inbox** | Желания (запросы) по проекту — ваш бэклог идей |
| **Tasks** | Спланированные и приоритизированные задачи, сгенерированные из желаний |
| **Terminal** | Предпросмотр вывода терминала (200 строк) с возможностью отправки клавиш |

### Определение статуса

| Статус | Индикатор | Паттерны |
|--------|-----------|----------|
| **Working** | Жёлтый | Спиннеры, «thinking», индикаторы работы инструментов |
| **Needs Input** | Синий | Промпт `>`, `?`, `(y/n)`, командная строка |
| **Error** | Красный | «Error:», «Failed», «rate limit» |
| **Idle** | Серый | Нет недавней активности |
| **Offline** | Тёмный | Сессия tmux не найдена |

## Установка

### Быстрый старт

```bash
git clone https://github.com/your-username/claude-session-manager.git
cd claude-session-manager/csm
bash ../start.sh
```

Скрипт `start.sh` проверяет зависимости, устанавливает npm-пакеты и запускает веб-дашборд.

### Ручная установка

```bash
cd csm
npm install
npm link          # делает команду 'csm' доступной глобально
```

### Настройка tmux (опционально)

Запустите интерактивный мастер настройки для конфигурации плагинов tmux и регистрации сессий проектов:

```bash
bash templates/setup.sh
```

Или примените рекомендуемую конфигурацию tmux вручную:

```bash
cp templates/tmux-csm.conf ~/.tmux-csm.conf
echo 'source-file ~/.tmux-csm.conf' >> ~/.tmux.conf
tmux source-file ~/.tmux.conf
```

## Использование

### Команды CLI

```bash
csm web                              # Запустить веб-дашборд (localhost:9847)
csm status                           # Показать статусы всех сессий
csm status --watch                   # Автообновление каждые 3 секунды

csm add <name> <tmux-session>        # Зарегистрировать сессию для мониторинга
csm add bms bms-session --dir ~/projects/bms
csm remove <name>                    # Удалить сессию из мониторинга
csm list                             # Список зарегистрированных сессий
csm discover                         # Найти доступные сессии tmux

csm send <name> <text>               # Отправить текст в сессию
csm focus <name>                     # Переключить фокус tmux на сессию
csm config --show                    # Показать текущую конфигурацию
```

### npm-скрипты

```bash
npm start        # Запуск CLI
npm run web      # Запуск веб-дашборда
npm run dev      # Веб-дашборд (режим разработки)
```

## Конфигурация

CSM хранит настройки в `~/.csm/config.json`:

```json
{
  "port": 9847,
  "pollInterval": 3000,
  "historyRetention": 30,
  "alerts": {
    "needsInputTimeout": 300,
    "idleTimeout": 600,
    "tokenThreshold": 80
  },
  "sessions": []
}
```

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `port` | `9847` | Порт веб-дашборда |
| `pollInterval` | `3000` | Интервал опроса в мс |
| `historyRetention` | `30` | Дней хранения истории |
| `alerts.needsInputTimeout` | `300` | Секунд до алерта «Needs Input» |
| `alerts.idleTimeout` | `600` | Секунд до алерта «Idle» |
| `alerts.tokenThreshold` | `80` | Порог алерта использования токенов (%) |

## API

Веб-сервер предоставляет REST-эндпоинты и WebSocket-соединение.

### REST-эндпоинты

**Сессии**
- `GET /api/sessions` — состояния всех сессий
- `GET /api/sessions/:name` — состояние одной сессии
- `POST /api/sessions/:name/send` — отправить текстовый ввод
- `POST /api/sessions/:name/focus` — переключить фокус tmux
- `POST /api/sessions/:name/keys` — отправить клавиши tmux
- `POST /api/sessions/create` — создать новую сессию
- `POST /api/sessions/:name/kill` — завершить сессию

**Пайплайн**
- `GET /api/pipeline/:name/wishes` — список желаний
- `POST /api/pipeline/:name/wishes` — создать желание
- `GET /api/pipeline/:name/tasks` — список задач
- `POST /api/pipeline/:name/tasks` — создать задачу
- `POST /api/pipeline/:name/plan` — AI-планирование (желания → задачи)
- `POST /api/pipeline/:name/execute` — выполнить следующую задачу

**История и алерты**
- `GET /api/history/:name/status` — история статусов (24ч)
- `GET /api/history/:name/tokens` — история использования токенов
- `GET /api/alerts` — неподтверждённые алерты

### WebSocket

Подключение к `ws://localhost:9847`. Сообщения в формате JSON с полем `type`:

- `update` — периодическая трансляция состояния
- `statusChange` — смена статуса сессии
- `alert` — сработал алерт
- `taskStarted` — начато выполнение задачи

## Пайплайн задач

CSM предоставляет структурированный рабочий процесс для управления работой по проектам:

1. **Желания (Inbox)** — запишите запросы и идеи для проекта
2. **Планирование** — AI анализирует желания и генерирует конкретные задачи
3. **Задачи** — приоритизированные, сфокусированные единицы работы
4. **Выполнение** — задачи выполняются по одной, каждая в своей сессии Claude

Этот пайплайн позволяет накапливать идеи, планировать их пакетно и выполнять систематически.
